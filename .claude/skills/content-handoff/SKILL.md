# Content Handoff

## Purpose
Use this skill to turn content work into a clean execution brief for another agent.

This skill helps:
- preserve writing intent during implementation
- clarify what must be used as written
- reduce ambiguity in downstream work
- align copy with layout and SEO needs
- improve coordination across agents

## When to use
Use this skill when:
- copy is ready for implementation
- design or frontend needs final text
- SEO needs content constraints respected
- marketing needs a content-aligned summary
- tech lead needs a clear content package

## Workflow

### 1. Define the target owner
Clarify:
- which agent should take the next step
- why this owner is the right one
- what they are expected to implement or review

### 2. Summarize the content objective
State:
- page or section goal
- audience
- key messages
- CTA or informational outcome
- important tone constraints

### 3. State execution requirements
Include:
- required text blocks
- fixed copy vs flexible copy
- SEO-sensitive content when relevant
- layout dependencies when relevant
- what should not drift

### 4. Define validation points
Specify:
- what must remain intact
- what should be checked before approval
- what should return to content for final review

## Output format
Always structure the answer as:

## Handoff target
## Objective
## Audience and content intent
## Required content elements
## Constraints
## Deliverables expected
## Validation points
## Return to content with

## Guardrails
- Do not hand off vague writing
- Keep fixed copy explicit
- Separate approved copy from draft areas
- Make validation criteria concrete
