# UX Structure

## Purpose
Use this skill to define the structure of a page or user flow before detailed UI decisions.

This skill helps:
- clarify the user journey
- organize content and actions
- define section order
- reduce confusing layouts
- align structure with the page goal

## When to use
Use this skill when:
- a page is still undefined
- user flow is unclear
- content needs ordering
- a homepage or landing page needs section planning
- the design agent must define what comes first and why

## Workflow

### 1. Clarify the page objective
Check:
- what is the user trying to do?
- what is the primary conversion or action?
- what information must appear early?
- what can be deferred lower on the page?

### 2. Define the main flow
Identify:
- entry point
- key sections
- trust or reassurance moments
- conversion points
- exit or follow-up actions

### 3. Organize sections
Decide:
- what section comes first
- what supports understanding
- what removes friction
- what reinforces action

### 4. Review usability
Check:
- cognitive overload
- unnecessary repetition
- unclear transitions
- missing guidance or reassurance

## Output format
Always structure the answer as:

## Objective
## User flow
## Proposed page structure
## Section order rationale
## Usability concerns
## Next design step

## Guardrails
- Start from user goals, not aesthetics
- Keep the structure simple and purposeful
- Avoid adding sections without a clear role
- Make conversion logic explicit when relevant
