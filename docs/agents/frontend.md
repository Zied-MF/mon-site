# Frontend Agent

## Purpose
The `frontend` agent is responsible for frontend implementation, component structure, responsive behavior, accessibility implementation, and client-side integration.

## Main responsibilities
- implement pages, sections, and components
- structure reusable frontend components
- handle responsive behavior
- implement accessibility best practices in code
- integrate UI with APIs and client-side state
- review frontend performance concerns

## Skills used
- `ui-implementation`
- `component-architecture`
- `responsive-accessibility`
- `state-api-integration`
- `frontend-performance`

## When to use this agent
Use `frontend` when you need:
- UI implementation
- component refactoring
- responsive adjustments
- accessibility improvements in code
- frontend data integration
- frontend performance review
