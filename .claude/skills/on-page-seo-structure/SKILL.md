# On-Page SEO Structure

## Purpose
Use this skill to define strong on-page SEO structure for a page before or during implementation.

This skill helps:
- organize headings and sections
- clarify title and meta direction
- improve content hierarchy
- align content blocks with search intent
- reduce weak on-page structure

## When to use
Use this skill when:
- a page needs SEO structure
- heading logic is unclear
- title and meta direction must be defined
- content sections need better hierarchy
- a page is being planned before implementation

## Workflow

### 1. Confirm page intent
Check:
- what the page is trying to rank for
- what must be explained early
- what subtopics must be covered
- what is unnecessary for this page

### 2. Define SERP-facing elements
Clarify:
- title direction
- meta description direction
- primary heading focus
- supporting heading hierarchy

### 3. Define content structure
Organize:
- intro/context
- core explanatory or commercial sections
- trust/supporting sections
- CTA or next-step content when relevant

### 4. Review optimization quality
Check:
- weak or generic headings
- repetitive structure
- thin support for the target intent
- poor alignment between title and page body

## Output format
Always structure the answer as:

## Objective
## Page intent
## Title direction
## Meta description direction
## Heading structure
## Recommended content blocks
## On-page risks
## Next implementation step

## Guardrails
- Keep headings useful for humans first
- Do not stuff keywords mechanically
- Align title, H1, and page body logically
- Prefer strong structure over forced optimization tricks
