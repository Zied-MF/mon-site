# Design Handoff

## Purpose
Use this skill to turn design decisions into a clean frontend-ready handoff.

This skill helps:
- clarify what frontend should build
- reduce ambiguity in implementation
- capture layout intent
- document important UX constraints
- improve execution consistency

## When to use
Use this skill when:
- design direction is approved
- frontend implementation should start
- layout and hierarchy have been defined
- the design agent must prepare a clear handoff
- important UX constraints must not be lost

## Workflow

### 1. Define the target implementation
Clarify:
- what page or section is being handed off
- what the frontend must build
- what the expected outcome is
- what is already decided

### 2. State scope
Include:
- required UI blocks
- expected interactions
- hierarchy expectations
- responsive intent
- accessibility expectations

### 3. State constraints
Clarify:
- what must remain consistent
- what is flexible
- what should not be changed by implementation
- what assumptions are still open

### 4. Define validation points
Specify:
- what must be checked before sign-off
- what would break the intended UX
- what should return to design for review

## Output format
Always structure the answer as:

## Handoff target
## Objective
## Scope
## UX and layout intent
## Constraints
## Responsive expectations
## Validation points
## Return to design with

## Guardrails
- Do not hand off vague design work
- Keep scope explicit
- Separate fixed decisions from flexible implementation details
- Make validation criteria concrete
