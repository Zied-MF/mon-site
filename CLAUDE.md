# Repository guidance

## Agent roles
- `tech-lead` is the default architecture and planning agent.
- Implementation should start only after a clear plan exists.

## Engineering principles
- Prefer simple and reversible decisions
- Keep components modular
- Avoid premature abstraction
- Document major decisions in `/docs/architecture/`

## Workflow
1. Ask `tech-lead` for framing
2. Validate approach
3. Delegate implementation to specialist agents
4. Review before merge
