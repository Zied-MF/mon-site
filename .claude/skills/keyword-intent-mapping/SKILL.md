# Keyword Intent Mapping

## Purpose
Use this skill to align pages with clear search intent and realistic keyword focus.

This skill helps:
- identify the main search intent
- separate primary and secondary targets
- avoid vague keyword targeting
- map pages to meaningful topics
- reduce cannibalization risk

## When to use
Use this skill when:
- a page needs SEO targeting
- search intent is unclear
- multiple keywords compete for one page
- topic mapping must be defined
- the repository needs page-to-query alignment

## Workflow

### 1. Clarify the page goal
Check:
- what the page is trying to achieve
- what the user wants when searching
- whether the page is informational, commercial, navigational, or transactional
- what outcome the searcher expects

### 2. Define intent clusters
Identify:
- primary intent
- secondary intent
- supporting subtopics
- queries that should not be targeted by this page

### 3. Map relevance
Clarify:
- what this page should rank for
- what should belong to another page
- what terms reinforce relevance without changing intent
- what risks cannibalization

### 4. Review fit
Check:
- mismatch between content and intent
- overbroad targeting
- weak differentiation between pages
- missing supporting context

## Output format
Always structure the answer as:

## Objective
## Primary search intent
## Secondary intent
## Recommended keyword focus
## What this page should not target
## Relevance risks
## Next SEO step

## Guardrails
- Prefer one clear primary intent per page
- Do not force many intents into one page
- Separate page targeting from broad topic ownership
- Keep intent aligned with what users actually want
