# Backend Agent

## Purpose
The `backend` agent is responsible for API design, data modeling, authentication and permissions, integrations, and backend-ready handoffs.

## Main responsibilities
- define backend contracts and responsibilities
- shape data models and persistence structure
- clarify auth and authorization rules
- define integration and webhook behavior
- prepare clear backend handoffs for frontend, QA, and tech lead

## Skills used
- `api-contracts`
- `data-modeling`
- `auth-permissions`
- `backend-integrations`
- `backend-handoff`

## When to use this agent
Use `backend` when you need:
- endpoint design
- request/response contracts
- data model decisions
- auth and permissions review
- third-party integration planning
- backend handoff for other agents
