# Design System Consistency

## Purpose
Use this skill to keep the interface consistent and reusable across pages and components.

This skill helps:
- define repeatable UI patterns
- reduce visual inconsistency
- support scalable interface decisions
- improve reuse across the product
- prepare cleaner frontend implementation

## When to use
Use this skill when:
- repeated UI patterns are emerging
- a design starts to feel inconsistent
- spacing, buttons, cards, or section patterns vary too much
- the team needs reusable design rules
- a design handoff should reflect shared UI conventions

## Workflow

### 1. Identify repeated patterns
Check:
- buttons
- cards
- section headers
- form patterns
- spacing patterns
- icon/text combinations

### 2. Define consistency rules
Clarify:
- when the same pattern should repeat
- what should vary
- what tokens or rules should exist conceptually
- where inconsistency creates UX friction

### 3. Review maintainability
Check:
- design drift risk
- pattern duplication
- inconsistent interaction expectations
- unnecessary exceptions

### 4. Prepare reuse
Decide:
- what should become a shared pattern
- what remains page-specific
- what should be documented for frontend

## Output format
Always structure the answer as:

## Objective
## Repeated patterns found
## Consistency rules
## Reuse opportunities
## Risks of inconsistency
## Recommended system direction

## Guardrails
- Prefer reusable patterns over one-off exceptions
- Keep the system proportional to project maturity
- Avoid inventing a heavy design system too early
- Focus on practical consistency first
