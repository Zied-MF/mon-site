# Wireframing Layout

## Purpose
Use this skill to turn a structured page idea into a layout proposal.

This skill helps:
- define layout zones
- organize content visually
- shape page sections
- prepare implementation-ready structure
- avoid vague design direction

## When to use
Use this skill when:
- a page structure already exists
- the next step is layout definition
- a section needs wireframing
- content blocks need visual arrangement
- frontend handoff needs more clarity

## Workflow

### 1. Identify layout needs
Clarify:
- what content elements must be shown
- what deserves visual priority
- what can be grouped
- what should stay visually separated

### 2. Define zones
Map:
- top-level container
- left / right or stacked areas
- supporting blocks
- CTA placement
- trust or proof areas

### 3. Review responsiveness conceptually
Check:
- what should stack on smaller screens
- what can stay side-by-side
- what spacing assumptions might break
- what layout relationships are essential

### 4. Prepare handoff clarity
Ensure:
- layout roles are understandable
- section composition is explicit
- major placement decisions are documented

## Output format
Always structure the answer as:

## Objective
## Layout zones
## Section composition
## Responsive layout intent
## Wireframe notes
## Handoff considerations

## Guardrails
- Keep wireframes structural, not decorative
- Prefer clear block organization over visual flourish
- Make layout relationships explicit
- Do not rely on unstated visual assumptions
