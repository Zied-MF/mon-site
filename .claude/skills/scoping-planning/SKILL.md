# Scoping Planning

## Purpose
Use this skill to turn a request into a clear execution plan.

This skill helps:
- clarify the real objective
- separate must-have from nice-to-have
- identify dependencies
- break work into phases
- create a realistic implementation sequence

## When to use
Use this skill when:
- a request is still vague
- a feature is too broad
- multiple tasks are mixed together
- priorities are unclear
- the tech lead needs to define what should happen first
- implementation should be phased instead of done all at once

## Workflow

### 1. Clarify the goal
Check:
- what is the user actually trying to achieve?
- what outcome matters most?
- what is in scope right now?
- what is explicitly out of scope?

### 2. Break down the work
Split the request into:
- core deliverables
- supporting tasks
- dependencies
- optional enhancements

### 3. Prioritize
Classify items as:
- must-have
- should-have
- nice-to-have
- later / out of scope

### 4. Sequence the work
Define:
- what must happen first
- what can happen in parallel
- what is blocked by other work
- what should be deferred

### 5. Produce an execution plan
The plan should:
- start with the minimum viable path
- reduce risk early
- keep ownership clear
- avoid unnecessary parallelization

## Output format
Always structure the answer as:

## Objective
## In scope
## Out of scope
## Work breakdown
## Priorities
## Dependencies
## Recommended sequence
## Execution plan
## Open questions

## Guardrails
- Prefer smaller phases over large ambiguous delivery
- Do not treat optional ideas as required scope
- Make trade-offs explicit
- Keep the first version as small as possible
- If dependencies are unclear, state assumptions explicitly
