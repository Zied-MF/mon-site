# Internal Linking Architecture

## Purpose
Use this skill to define meaningful internal linking and page relationships.

This skill helps:
- connect related pages clearly
- strengthen topic architecture
- improve crawl paths
- reduce orphan content risk
- support relevance distribution across the site

## When to use
Use this skill when:
- multiple pages or sections exist
- the site structure needs planning
- page relationships are unclear
- topic clusters should be organized
- internal linking opportunities need definition

## Workflow

### 1. Map page roles
Check:
- which pages are core
- which pages support them
- which pages should link to each other
- what pages may become isolated

### 2. Define linking logic
Clarify:
- parent-child relationships
- sibling relationships
- contextual linking opportunities
- navigation vs editorial linking roles

### 3. Review crawl and relevance flow
Check:
- whether key pages are easy to reach
- whether support pages reinforce the right targets
- whether anchors should be more specific
- whether the structure feels flat or fragmented

### 4. Reduce architecture risks
Check:
- orphan pages
- vague anchor logic
- unnecessary deep nesting
- weak cluster relationships

## Output format
Always structure the answer as:

## Objective
## Core pages and supporting pages
## Internal linking logic
## Cluster or architecture proposal
## Risks detected
## Recommended next step

## Guardrails
- Link with purpose, not volume alone
- Keep anchor logic clear and relevant
- Make core pages easier to discover than peripheral ones
- Prefer understandable architecture over artificial complexity
