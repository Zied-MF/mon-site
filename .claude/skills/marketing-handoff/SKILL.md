# Marketing Handoff

## Purpose
Use this skill to turn marketing direction into a clean execution brief for another agent.

This skill helps:
- clarify what should be executed next
- preserve strategic intent
- reduce ambiguity in implementation
- align downstream work with conversion goals
- improve coordination across agents

## When to use
Use this skill when:
- marketing direction is approved
- design or frontend should implement a page
- SEO needs message priorities
- content work needs a clear brief
- multiple agents depend on the same messaging logic

## Workflow

### 1. Define the target owner
Clarify:
- which agent should take the next step
- why this owner is the right one
- what they are expected to produce

### 2. Summarize the marketing objective
State:
- the audience
- the main conversion goal
- the offer
- the core message
- the trust logic

### 3. State execution constraints
Include:
- required messages
- forbidden drift
- CTA expectations
- audience assumptions
- dependencies on design or tech decisions

### 4. Define validation points
Specify:
- what must remain intact
- what should be checked before approval
- what must come back for marketing review

## Output format
Always structure the answer as:

## Handoff target
## Objective
## Audience and offer context
## Required messages
## CTA and conversion intent
## Constraints
## Deliverables expected
## Validation points
## Return to marketing with

## Guardrails
- Do not hand off vague messaging
- Keep the conversion goal explicit
- Separate fixed messaging from flexible implementation details
- Make validation criteria concrete
