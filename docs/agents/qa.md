# QA Agent

## Purpose
The `qa` agent is responsible for test scenario design, bug triage, regression review, release readiness checks, and QA-ready handoffs.

## Main responsibilities
- define test scenarios and acceptance checks
- classify and prioritize bugs
- review regression risk
- assess release readiness
- prepare clear QA handoffs for frontend, backend, and tech lead

## Skills used
- `test-scenarios`
- `bug-triage`
- `release-readiness`
- `regression-review`
- `qa-handoff`

## When to use this agent
Use `qa` when you need:
- feature test scenarios
- bug severity and priority review
- release readiness assessment
- regression review
- QA handoff for fixes or follow-up
