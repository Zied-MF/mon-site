# Visual Hierarchy

## Purpose
Use this skill to improve clarity, emphasis, and scanability in UI decisions.

This skill helps:
- prioritize information
- make CTAs visible and understandable
- improve readability
- reduce clutter
- guide the user’s eye through the interface

## When to use
Use this skill when:
- a page feels visually unclear
- too many elements compete for attention
- CTA prominence is uncertain
- copy and layout need clearer emphasis
- a section must become easier to scan

## Workflow

### 1. Identify priority levels
Clarify:
- what is the primary message?
- what is secondary?
- what is supportive?
- what should visually recede?

### 2. Review scanning flow
Check:
- what users see first
- what interrupts the reading path
- whether the CTA appears at the right time
- whether the section feels noisy

### 3. Improve emphasis
Adjust conceptually:
- headline strength
- spacing and grouping
- visual contrast
- CTA importance
- supporting content density

### 4. Reassess clarity
Check:
- ambiguity
- clutter
- weak CTA placement
- unclear reading order

## Output format
Always structure the answer as:

## Objective
## Information priority
## Scanning flow
## Hierarchy problems detected
## Recommended adjustments
## Expected UX impact

## Guardrails
- Prioritize clarity over decoration
- Do not make every element compete equally
- Keep CTA logic explicit
- Reduce noise before adding new UI treatments
