# Handoff Spec

## Purpose
Use this skill to prepare a clean and actionable handoff.

This skill helps:
- define who should take the next step
- package the task clearly
- reduce ambiguity before delegation
- make dependencies explicit
- improve execution quality across agents

## When to use
Use this skill when:
- the tech lead has finished framing a task
- a specialist agent needs a clear brief
- implementation should start soon
- multiple dependencies must be clarified before handoff
- ownership is known but execution details still need structure

## Workflow

### 1. Define the target owner
State:
- which agent should take the task
- why this owner is the right one
- what this owner is responsible for
- what remains outside this owner's scope

### 2. Summarize the objective
Clarify:
- the outcome to achieve
- the deliverable expected
- the success criteria
- any important context already decided

### 3. State constraints
Include:
- technical constraints
- design constraints
- SEO or marketing constraints when relevant
- timeline or sequencing constraints
- repository conventions to follow

### 4. List dependencies
Clarify:
- what must already exist
- what is still missing
- what blocks execution
- what can be assumed for now

### 5. Define the execution brief
The handoff should include:
- what to do
- what not to do
- what to validate before completion
- what to return back to the tech lead

## Output format
Always structure the answer as:

## Handoff target
## Objective
## Scope
## Out of scope
## Constraints
## Dependencies
## Deliverables expected
## Validation points
## Return to tech lead with

## Guardrails
- Do not hand off vague work
- Keep ownership explicit
- Distinguish scope from assumptions
- State blockers clearly
- Make success criteria concrete
- Prefer short and actionable handoffs over long generic instructions
