# SEO Handoff

## Purpose
Use this skill to turn SEO direction into a clean execution brief for another agent.

This skill helps:
- preserve SEO intent during execution
- clarify what another agent must implement
- reduce ambiguity across teams
- keep search priorities visible
- improve coordination between SEO and implementation

## When to use
Use this skill when:
- SEO direction is approved
- another agent must implement a page or change
- content and structure requirements must be passed on
- frontend needs SEO constraints
- design or marketing needs search priorities clarified

## Workflow

### 1. Define the target owner
Clarify:
- which agent should take the next step
- why this owner is the right one
- what they are expected to deliver

### 2. Summarize the SEO objective
State:
- page intent
- primary target
- supporting relevance goals
- internal linking role
- SEO-critical constraints

### 3. State execution requirements
Include:
- required page elements
- title/meta direction
- heading expectations
- linking expectations
- things that should not drift

### 4. Define validation points
Specify:
- what must be checked before approval
- what would break the intended SEO logic
- what should return to SEO for review

## Output format
Always structure the answer as:

## Handoff target
## Objective
## Search intent and target
## Required SEO elements
## Constraints
## Deliverables expected
## Validation points
## Return to SEO with

## Guardrails
- Do not hand off vague SEO guidance
- Keep the primary page intent explicit
- Separate fixed SEO requirements from flexible implementation details
- Make validation criteria concrete
